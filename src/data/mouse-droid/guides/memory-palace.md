The Memory Palace is the local Obsidian vault where Mouse Droid's narration lives. It's
just Markdown notes under the hood, which means you can open, edit, and link them yourself
— but a little structure goes a long way. Here's how to get the most out of it.

### Recall by meaning, not keywords

Recall is **hybrid search**: it blends classic keyword matching with semantic
(meaning-based) search and fuses the two into one ranked result. So you can ask for *"the
thing about the flaky deploy"* and get the right note even if you never wrote the word
"flaky." When you ask the droid a question in Chat, this is what he reaches for when your
recent activity alone doesn't have the answer.

### Lean on hubs

Every project gets a **hub** note that gathers its sessions, summaries, and related notes
in one place. When you're reloading context after time away, start at the hub rather than
hunting through individual entries — it's the front door to a project. If you've enabled
hub enrichment, the droid keeps a rolling summary there so the hub stays a useful at-a-
glance picture.

### Trust the provenance

Mouse Droid writes its memories in **sessions**, and every entry it creates is stamped with
where it came from — marked as authored by Mouse Droid and tied to the session that
produced it, with time entries linked back to their session too. Before adding something
new, it even checks whether a near-duplicate already exists. So when a memory looks off,
you can trace it straight back to the work behind it instead of guessing — and because the
notes are plain Obsidian, you're always free to correct them yourself.
