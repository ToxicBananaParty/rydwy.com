When you need to tell someone (or yourself) what happened, Reports writes it up. There are
two styles, and you pick the time range each covers — today, the last seven days, and so on.

**Daily Summary** is plain prose, grouped by project — a readable recap of what you worked
on over the chosen period.

**Top 5 Things** is an executive-style status for a single project, sorted into five
buckets: what's **working**, what's **broken**, what's **risky**, what's **surprising**,
and what **needs a decision**. It's built to paste into a standup or a status update.

### Sharing them

Each report has **Google Chat** and **Email** buttons. Both copy a clean, plain-text
version to your clipboard so you can paste it wherever it needs to go; the Google Chat
button also opens Google Chat in your browser to save you a step.

### Project descriptions

The project pills at the top of each report carry a small **✎** button. Tap it to add
or edit a **description** for that project — a sentence or two about the apps, sites,
and topics that belong to it. The description sharpens how the classifier assigns
activity to the project and gives the report generator richer context to write from.
You can also set or update a description by editing the opening paragraph of the
project's hub note in your vault.

### Neutral voice

By default reports carry the droid's voice. Flip the **neutral-voice toggle** and the
persona drops away for a straight, professional tone — better when the report is going to
colleagues. You can set which way that toggle starts in Settings.

### Regenerate with feedback

Every report has a **Regenerate** button. Click the caret beside it to add a short note —
a correction or bit of context — before regenerating. If a summary assumed that an hour in
your browser was recreation when it was research, tell it so and regenerate; the note is
sent along with that one request. It's one-shot: it isn't saved, and the next plain
regenerate goes back to the normal report.

_Find it in the **Reports** tab._
